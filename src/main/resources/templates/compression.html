Message Compression

Kafka поддерживает сжатие сообщений на уровне Producer-а. Сжатие уменьшает размер данных на диске и в сети, но увеличивает нагрузку на CPU. Сравним 5 режимов на реальных JSON-сообщениях.

Как работает сжатие

Producer сжимает batch сообщений перед отправкой. Broker хранит данные в сжатом виде. Consumer распаковывает при чтении.

Producer → сжатие → Broker → распаковка → Consumer
Когда использовать
  • LZ4 / Snappy — высокая пропускная способность, низкая задержка
  • ZSTD — лучший баланс размера и скорости
  • GZIP — максимальное сжатие, если CPU не bottleneck
  • Без сжатия — минимальная нагрузка на CPU
Настройка

Одна строка в конфигурации Producer:

compression.type=lz4

Значения: none, gzip, snappy, lz4, zstd

Параметры теста

Каждое сообщение — реалистичный JSON (~400 байт): event tracking с userId, url, userAgent, tags и metadata.
