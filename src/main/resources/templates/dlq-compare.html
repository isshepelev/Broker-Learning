Dead Letter Queue: Kafka vs RabbitMQ

Отправьте "плохое" сообщение в оба брокера и наблюдайте, как каждый обрабатывает ошибки. Слова poison, bad, error, fail, crash, broken вызывают ошибку обработки.

Kafka DLT (Dead Letter Topic)
Producer
→
Topic

Consumer
↻ 3 retry (1 сек)

DLT

Механизм: Spring Kafka DefaultErrorHandler перехватывает исключение, повторяет 3 раза с интервалом 1 сек. Если все попытки неудачны — DeadLetterPublishingRecoverer отправляет сообщение в topic с суффиксом .DLT.

  • Настраиваемое число retry и интервал
  • Сообщение сохраняется в DLT topic навсегда
  • Можно перечитать и переотправить
  • Медленнее: retry занимают время
RabbitMQ DLX (Dead Letter Exchange)
Producer
→
Queue
→
Consumer
reject (no requeue)
→
DLQ

Механизм: Consumer бросает AmqpRejectAndDontRequeueException — RabbitMQ мгновенно отправляет сообщение в Dead Letter Exchange (DLX), откуда оно попадает в Dead Letter Queue.

  • Мгновенный reject без retry
  • Сообщение удаляется из основной очереди
  • DLX + routing rules = гибкая маршрутизация ошибок
  • Быстрее: нет ожидания retry
