Ordering гарантии в Kafka

Kafka гарантирует порядок сообщений только внутри одной партиции. Если у топика несколько партиций — глобальный порядок теряется. Это ключевая концепция для проектирования любого Kafka-приложения.

1 партиция = строгий порядок

Все сообщения хранятся в одном логе. Consumer читает их ровно в порядке отправки.

1 → 2 → 3 → 4 → 5
Читаем: 1, 2, 3, 4, 5 — порядок идеальный
5 партиций без ключа = хаос

Без ключа сообщения распределяются round-robin. При чтении партиции читаются последовательно — порядок ломается.

P014
P125
P23
Читаем: 1, 4, 2, 5, 3 — порядок нарушен!
Ключ = решение

Сообщения с одним ключом всегда попадают в одну партицию. Порядок для каждого ключа (пользователя, заказа) гарантирован.

P0A1→A2→A3
P2B1→B2→B3
A: 1→2→3, B: 1→2→3 — оба в порядке
Демо 1: Без ключа — round-robin распределение

Отправим пронумерованные сообщения без ключа в два топика: с 1 партицией и с 5 партициями. В первом порядок сохранится, во втором — нет.

Демо 2: С ключами — порядок внутри ключа

Отправим сообщения с разными ключами в топик с 5 партициями. Каждый ключ попадёт в свою партицию — и порядок внутри ключа сохранится. Это решение проблемы из Демо 1.
