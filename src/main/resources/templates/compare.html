Kafka vs RabbitMQ

Отправьте одно и то же сообщение в оба брокера и наглядно увидьте разницу в их работе.

K
Apache Kafka
Распределённый лог событий
  • 📋 Лог — сообщения хранятся на диске и не удаляются после чтения
  • 🔁 Replay — можно перечитать данные с любого offset
  • ⚡ Throughput — миллионы msg/sec, sequential I/O
  • 👥 Consumer Groups — несколько групп читают независимо
  • 📚 Partitions — параллелизм через партиционирование
  • 🕐 Retention — хранение 7 дней / навсегда по настройке
R
RabbitMQ
Классический message broker
  • 📩 Очередь — сообщение удаляется после подтверждения consumer-ом
  • ❌ Нет replay — прочитанное сообщение исчезает навсегда
  • 🔌 Routing — гибкая маршрутизация через exchanges и bindings
  • ✅ Acknowledgment — надёжная доставка с подтверждением
  • 🔨 Протоколы — AMQP, MQTT, STOMP
  • 📋 Простота — легко настроить для задач request/reply
