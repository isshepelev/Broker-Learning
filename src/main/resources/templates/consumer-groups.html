Consumer Groups

Узнайте, как Kafka организует consumer-ов в группы для параллельной обработки и отказоустойчивости.

Что такое Consumer Group?

Consumer group -- набор consumer-ов, совместно обрабатывающих данные из topic-ов. Каждый partition назначается ровно одному consumer-у в группе. При сбое Kafka перебалансирует partition-ы между оставшимися участниками.

Назначение Partition-ов
  • Каждый partition обрабатывается ровно одним consumer-ом в группе
  • Один consumer может обрабатывать несколько partition-ов
  • Если consumer-ов больше чем partition-ов, часть будет простаивать
  • Для максимального параллелизма: число consumer-ов = число partition-ов

Совет: несколько consumer group-ов могут независимо читать из одного topic-а.

Rebalancing

Rebalancing происходит при изменении состава группы:

  • Новый consumer: partition-ы перераспределяются
  • Consumer выходит: его partition-ы переназначаются
  • Добавлены partition-ы: запускается rebalancing
Стратегии:
  • Eager: все отдают partition-ы и заново присоединяются
  • Cooperative: только затронутые partition-ы перераспределяются
Consumer Groups в кластере

Consumer group-ов не найдено. Они создаются автоматически при подписке на topic-и.

  • group-id Consumer Group
Управление Offset-ами
Автоматический коммит

С enable.auto.commit=true offset-ы коммитятся автоматически через интервал auto.commit.interval.ms. Просто, но может привести к потерям (at-most-once).

Ручной коммит

С enable.auto.commit=false приложение явно коммитит offset-ы. Обеспечивает at-least-once. Используйте commitSync() или commitAsync().
