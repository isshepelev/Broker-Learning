Consumer Groups

Kafka организует consumer-ов в группы для параллельной обработки и отказоустойчивости. Здесь вы видите реальные группы вашего кластера, их участников, назначенные партиции, offset-ы и lag.

Назначение партиций
  • Каждый partition — ровно одному consumer-у
  • Один consumer может читать несколько partition-ов
  • Несколько групп независимо читают один topic
Offset и Lag
  • Current offset — до куда consumer прочитал
  • End offset — последнее сообщение в партиции
  • Lag = end − current (сколько не обработано)
Состояния группы
  • Stable — все партиции распределены
  • PreparingRebalance — идёт перебалансировка
  • Empty — нет активных consumer-ов
  • Dead — группа удалена
