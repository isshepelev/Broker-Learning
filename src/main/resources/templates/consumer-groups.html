Consumer Groups

Узнайте, как Kafka организует consumer-ов в группы для параллельной обработки и отказоустойчивости.

Что такое Consumer Group?

Consumer group -- это набор consumer-ов, которые совместно обрабатывают данные из topic-ов. Каждый partition назначается ровно одному consumer-у в группе, что обеспечивает параллелизм. Если consumer выходит из строя, Kafka перебалансирует его partition-ы между оставшимися участниками группы.

Назначение Partition-ов

При присоединении consumer-ов к группе Kafka распределяет partition-ы между ними:

  • Каждый partition обрабатывается ровно одним consumer-ом в группе
  • Один consumer может обрабатывать несколько partition-ов
  • Если consumer-ов больше, чем partition-ов, часть consumer-ов будет простаивать
  • Для максимального параллелизма число consumer-ов должно быть равно числу partition-ов

Совет: несколько consumer group-ов могут независимо читать из одного topic-а. Каждая группа отслеживает свои offset-ы.

Rebalancing

Rebalancing происходит при изменении состава группы:

  • Новый consumer присоединяется: partition-ы перераспределяются
  • Consumer уходит или падает: его partition-ы переназначаются
  • Добавлены новые partition-ы: запускается rebalancing
Стратегии Rebalancing-а
  • Eager (по умолчанию): все consumer-ы отдают все partition-ы и заново присоединяются. Кратковременная остановка обработки.
  • Cooperative (инкрементальный): только затронутые partition-ы перераспределяются. Минимизирует простой.
Consumer Groups в кластере

Consumer group-ов не найдено. Они создаются автоматически, когда consumer-ы подписываются на topic-и.

Управление Offset-ами
Автоматический коммит Offset

С enable.auto.commit=true offset-ы коммитятся автоматически через интервал auto.commit.interval.ms. Просто в использовании, но может привести к потере данных (at-most-once), если consumer упадёт до обработки сообщения, чей offset уже зафиксирован.

Ручной коммит Offset

С enable.auto.commit=false приложение явно коммитит offset-ы после успешной обработки. Это обеспечивает at-least-once доставку. Используйте commitSync() для блокирующего коммита или commitAsync() для неблокирующего.
